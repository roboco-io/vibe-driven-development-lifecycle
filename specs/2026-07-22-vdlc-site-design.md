# VDLC 사이트 (vdlc.roboco.io) 설계 문서

- 날짜: 2026-07-22
- 상태: 승인됨 (심층 인터뷰 기반)
- 원본 아이디어: `vdlc.md`

## 1. 목적과 독자

**이중 목적**: 대외 매니페스토(로보코의 방법론 브랜드, 잠재 고객·CTO 대상 설득 서사) + 실무자용 실행 가이드(개발자/팀 리드가 바로 따라할 수 있는 실천서).

랜딩은 선언문으로 설득하고, 하위 페이지에서 실무 가이드로 확장하는 구조.

## 2. 핵심 결정 사항 (인터뷰 확정)

| 항목 | 결정 |
|---|---|
| 목적/독자 | 매니페스토 + 실무 가이드 이중 구조 |
| 도구 입장 | 서술은 도구 중립, 예시는 Claude Code 기준 |
| 콘텐츠 범위 | 6단계별 플레이북 + 템플릿 모음 + 도입 사례 + 성숙도 모델/지표 (전부) |
| 사례 방식 | 실제 경험 기반 익명화 (수치·상황은 작성 단계에서 검수) |
| 사례 소스 | ① 기업 컨설팅 경험 ② 로보코 자체 개발 경험 |
| 언어 | 한국어 우선, 추후 영어/일본어 확장 (VitePress i18n 관례 유지) |
| 배포 기술 | VitePress + GitHub Actions → GitHub Pages |
| 성숙도 모델 | 4레벨 (탐색–실천–자산화–표준화), 도입 경로 4단계와 정합 |

### 플레이북 페이지 공통 구조

각 `/guide/*` 페이지: ① 이 단계의 목적 ② 실행 순서 ③ 산출물 (템플릿 링크)
④ 완료 기준 체크리스트 ⑤ 흔한 실수 ⑥ "Claude Code로 하면" 박스 (도구별 구체 예시).

## 4. 성숙도 모델 (4레벨)

| 레벨 | 이름 | 특징 | 판별 신호 |
|---|---|---|---|
| 1 | 탐색 (Exploring) | 개인이 바이브코딩 실험 중, 프로세스 없음 | 프롬프트 세션마다 휘발, 검증은 "돌아가면 OK" |
| 2 | 실천 (Practicing) | 개인/소수가 VDLC 사이클을 돌림 | 의도 문서 작성 시작, 컨텍스트 파일 존재, 계획 승인 관문 사용 |
| 3 | 자산화 (Compounding) | 팀 단위 컨텍스트 자산 축적·재사용 | 팀 공용 규칙/스킬/템플릿, 리스크 기반 검증, 환류 의례 정착 |
| 4 | 표준화 (Systematizing) | 조직 표준 정착, 지표로 관리 | 리드타임·재작업률 측정, 온보딩에 자산 재사용 |

- 각 레벨에 "다음 레벨로 가는 열쇠" 1문장 포함 (예: L1→L2 = "첫 의도 문서 쓰기").
- 진단: 5원칙별 3~4문항 셀프 체크리스트 → 레벨 매핑.
- 측정 지표 (성숙도와 분리, 4종): 사이클 리드타임, 재작업률, 검증 통과율(1회 통과 비율), 컨텍스트 자산 증가량.

## 5. 배포 구성

- 리포지토리: `roboco-io/vibe-driven-development-lifecycle` (serithemage 계정 라우팅).
- VitePress 루트 = `docs/`. 마크다운 초안이 그대로 배포 소스.
- GitHub Actions → GitHub Pages, `CNAME`에 `vdlc.roboco.io`.
- DNS: `vdlc.roboco.io` → `roboco-io.github.io` CNAME 레코드는 사용자가 수동 등록.
- i18n: 한국어를 루트에 두고, 추후 `en/`, `ja/` 디렉토리 추가로 확장.

```
├── vdlc.md                      # 원본 아이디어 문서 (보존)
├── specs/                       # 설계·계획 문서 (사이트 미노출)
├── docs/                        # VitePress 루트
│   ├── index.md                 # 랜딩
│   ├── manifesto.md
│   ├── guide/ templates/ cases/ adoption/
│   └── .vitepress/config.ts
└── .github/workflows/deploy.yml
```

## 6. 작업 순서

1. **마크다운 초안**: `docs/` 아래 전체 페이지 초안 작성 (사례 페이지는 수치·상황 검수 요청 포함).
2. **검토·확정**: 사용자 리뷰 → 수정 반영.
3. **VitePress 세팅**: config, 테마(랜딩 히어로/카드), 사이드바, mermaid 렌더링.
4. **배포**: GitHub 리포 생성, Actions 워크플로, Pages + 커스텀 도메인 설정.
