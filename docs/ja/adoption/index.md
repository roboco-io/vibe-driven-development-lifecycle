# 導入

VDLCを組織に取り入れる過程を三つの文書に分けて整理した。いまチームがどのあたりにいるかを見積もり、次の一歩を定め、その一歩が実際に効果を出しているか確認する順で読む。

| 文書 | 何を扱うか |
|---|---|
| [成熟度モデル](/ja/adoption/maturity) | 4レベル(探索–実践–資産化–標準化)の判別基準と18問のセルフ診断チェックリスト。チームがいまどこにいるかからまず確認する。 |
| [導入ロードマップ](/ja/adoption/roadmap) | パイロット → コンテキスト資産化 → チーム拡散 → 組織標準、4つのステップの目標・やること・完了信号。成熟度レベルとステップが互いに対応する。 |
| [測定指標](/ja/adoption/metrics) | サイクルリードタイム、手戻り率、検証通過率、コンテキスト資産の増加量、4種の定義・測定法・解釈・ゲーミング防止。 |

初めてなら、[成熟度モデル](/ja/adoption/maturity)のセルフ診断で始めていまのレベルを確認したうえで、[導入ロードマップ](/ja/adoption/roadmap)でそのレベルに合った次の一歩を探す順を勧める。
